# Game Design Document Workflow

This folder is the mechanics-side companion to the lore bible.

Use this GDD the same way as the lore bible workflow:

1. Read `GDD_INDEX.md` first.
2. Pick one numbered file to work on.
3. The assistant asks numbered questions.
4. You answer with letters, short text, or corrections.
5. The assistant drafts clean prose inline.
6. Nothing becomes final until you say `lock in`.
7. Locked decisions get committed into the relevant numbered file.

## Hard Rules

- Do not infer missing mechanics.
- Do not invent systems without asking.
- Keep lore in the lore bible and mechanics in the GDD.
- Use plain Markdown only.
- Do not use emojis.
- Do not use italics.
- Use bold only for labels, locked terms, and important distinctions.
- Keep every open question visible until answered.
- When a mechanic touches lore, link back to the lore bible rather than duplicating lore prose.

## File Convention

Files use zero-padded numeric prefixes and snake_case names.

The index is the onboarding and portability hub. The numbered files are the working design documents.
