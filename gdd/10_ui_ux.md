# 10 UI and UX

## Purpose

Define screens, interaction flow, information hierarchy, onboarding, and accessibility.

## Status

**Design State:** Unlocked template.

**Last Locked Update:** None.

## Screen Map

### Main Menu

Define start, continue, settings, saves, and mode access.

### Campaign Map

Define region navigation, mission selection, rewards, and threat display.

### Party Management

Define hero roster, units, equipment, traits, upgrades, and comparison tools.

### Battle Preparation

Define formation setup, enemy preview, item assignment, and ready state.

### Combat HUD

Define health, energy, timers, statuses, trait activations, damage numbers, and battle log.

### Results Screen

Define rewards, stats, progression, unlocks, and next actions.

## UX Principles

### Readability

Define how the player reads threats, synergies, stats, and outcomes.

### Speed

Define how quickly players can start another fight or change builds.

### Onboarding

Define tutorials, tooltips, first battle structure, and complexity ramp.

### Accessibility

Define text size, color contrast, input remapping, motion settings, and readability needs.
