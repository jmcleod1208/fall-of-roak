# Lore Bible

This folder is the single source of truth for the game's story and world. Files are numbered to suggest reading order, but you can fill them out in any order.

## How to use this with AI coding assistants

When you ask your AI assistant to generate quests, NPC dialogue, item descriptions, or anything story-flavored, point it at the relevant file(s) in this folder. Examples:

- "Using `lore/04_characters.md`, write the dialogue tree for Eldra when the player first meets her."
- "Using `lore/06_systems.md`, design 5 starter spells consistent with the magic system."
- "Using `lore/03_factions.md` and `lore/05_locations.md`, generate a side quest involving the Iron Pact in Ashfall."

The more your assistant references these files, the more consistent your game's world will feel.

## File guide

| File | Purpose |
|------|---------|
| `00_overview.md` | Pitch, tone, themes — start here |
| `01_world.md` | Setting, geography, cosmology |
| `02_history.md` | Timeline and past events |
| `03_factions.md` | Groups, governments, guilds |
| `04_characters.md` | Major NPCs |
| `05_locations.md` | Cities, dungeons, regions |
| `06_systems.md` | Magic, tech, religion, economy |
| `07_bestiary.md` | Creatures and races |
| `08_items_artifacts.md` | Legendary items |
| `09_plot.md` | The main story |
| `10_glossary.md` | Terms and naming |

## Workflow tips

- **Fill `00_overview.md` first.** If the pitch isn't clear, nothing else will be.
- **Then sketch `01_world.md` and `06_systems.md`.** These are the rules everything else depends on.
- **Don't try to fill everything in one sitting.** Rough out skeletons across all files, then deepen iteratively.
- **Keep an `ideas.md` or `/scratch` folder for unfinalized concepts** so this folder stays canonical.
- **Commit changes regularly** so you can roll back if a creative direction doesn't work.
